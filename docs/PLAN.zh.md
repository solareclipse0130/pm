# 项目计划

本计划将项目管理 MVP 拆分为多个可审阅阶段。每个阶段都应完成、测试并总结后，再进入下一阶段。不要添加 `AGENTS.md` 中未描述的额外产品功能。

## 当前决策

- 前端：NextJS 静态构建，由 FastAPI 在 `/` 路径提供服务。
- 后端：Python FastAPI。
- 打包方式：一个用于本地运行的 Docker 容器。
- Docker 中的 Python 包管理器：`uv`。
- 数据库：本地 SQLite 数据库，如果缺失则自动创建。
- 认证：MVP 使用硬编码凭据 `user` 和 `password`，但数据库设计需要支持未来多用户。
- AI 提供商：DeepSeek 原生 API。
- AI 环境变量：从项目根目录 `.env` 读取 `DEEPSEEK_API_KEY`。
- AI 模型：`deepseek-v4-pro`。
- 现有前端：`frontend/` 已经包含一个可工作的纯前端 Kanban 演示，使用 NextJS、React、Vitest 和 Playwright。
